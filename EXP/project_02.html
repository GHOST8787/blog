AI 助理框架部署 (上)：
VM 連線與 Clawdbot 核心安裝

Tech & Auto Azure VM Clawdbot
2026.02.19 BY GHOST_1010

這是一份「新手友好」的說明書。上篇將帶你從環境準備開始，一步步在雲端 VM 上建立並啟動 Clawdbot 核心服務。🦞

# 第一步：環境準備

Clawdbot 是一個強大的 AI 助理框架，可以安裝在雲端 VM 上 24/7 運行。開始前，你需要準備以下項目：

  • 一個 雲端 VM（例如 Azure VM，需有對外 IP）
  • 一個 SSH 金鑰檔案（<YOUR_KEY.pem>）
  • 本機電腦（Windows，已安裝 PowerShell）

# 第二步：連線到 VM

在本機電腦的 PowerShell 中，切換到金鑰所在目錄並進行連線。成功後會看到 Ubuntu 的歡迎訊息。

cd <YOUR_LOCAL_KEY_PATH>
ssh -i <YOUR_KEY.pem> <YOUR_VM_USER>@<YOUR_VM_IP>

# 第三步：安裝 Clawdbot

使用 npm 全域安裝最新版 Clawdbot，並設定 gateway 服務讓它在背景常駐：

# 1. 安裝最新版 Clawdbot
sudo npm install -g clawdbot@latest

# 2. 確認安裝成功 (應會看到版本號)
clawdbot --version

# 3. 建立並啟動服務
clawdbot gateway setup --user
systemctl --user start clawdbot-gateway.service

# 4. 檢查服務狀態 (應顯示 active (running))
systemctl --user status clawdbot-gateway.service

# 第四步：解決 Gateway 埠口衝突（選用）

如果您發現服務狀態一直重啟，查看日誌若發現 Port 18789 is already in use，表示有舊程序佔用。請執行以下清理動作：

# 1. 停止所有相關服務與舊版檔案
systemctl --user stop clawdbot-gateway.service
systemctl --user stop openclaw-gateway.service 2>/dev/null
systemctl --user disable openclaw-gateway.service 2>/dev/null
rm -f ~/.config/systemd/user/openclaw-gateway.service

# 2. 殺光所有佔用 18789 埠的程序
sudo kill -9 $(sudo lsof -t -i:18789) 2>/dev/null

# 3. 重新載入並啟動乾淨的服務
systemctl --user daemon-reload
systemctl --user reset-failed clawdbot-gateway.service
systemctl --user start clawdbot-gateway.service

# 第五步：從本機電腦連線 Control UI

在本機開「新的」PowerShell 視窗，建立 SSH 隧道。注意：此視窗會卡住是正常的，請保持開啟。

ssh -i <YOUR_KEY.pem> -N -L 18789:127.0.0.1:18789 <YOUR_VM_USER>@<YOUR_VM_IP>

接著回到 VM 的終端機，取得 Dashboard 網址與 Token：

clawdbot dashboard

複製輸出的網址，將 127.0.0.1 改為 localhost，貼到瀏覽器即可開啟控制介面。若遇到 Token 錯誤，可透過 clawdbot config set gateway.auth.token <TOKEN> 重新設定。
