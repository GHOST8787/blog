AI 助理框架部署 (下)：
Google Workspace 整合與實戰

Tech & Auto Google Workspace OAuth
2026.02.19 BY GHOST_1010

接續上篇完成 Clawdbot 核心部署後，下篇將著重於安裝 Gog 技能並連接 Google 帳號，讓你的 AI 助理正式接管 Gmail 與行事曆。

# 第一步：安裝 Gog 技能

Gog 技能需要一個獨立的命令列工具，請在 VM 終端機執行以下指令進行安裝與配置：

# 1. 安裝 Gog CLI
cd /tmp
curl -fsSL https://api.github.com/repos/steipete/gogcli/releases/latest \
  | grep browser_download_url \
  | grep linux_amd64 \
  | cut -d '"' -f 4 \
  | xargs curl -fsSL -o gogcli.tgz

tar -xzf gogcli.tgz
sudo install -m 0755 gog /usr/local/bin/gog

# 2. 安裝 Clawdbot 技能
clawdbot skills install steipete/gog

# 第二步：Google OAuth 授權

為了安全連接您的 Gmail 與日曆，我們需要透過 Google Cloud 設定 OAuth 憑證：

  • 在 Google Cloud Console 建立專案並啟用 API (Gmail, Calendar, Drive 等)。
  • 建立「桌面應用程式」類型的 OAuth 用戶端 ID，並下載 JSON。

將下載的憑證透過 SCP 上傳到 VM：

scp -i <YOUR_KEY.pem> client_secret.json <YOUR_VM_USER>@<YOUR_VM_IP>:~/

在 VM 中完成授權配置：

gog auth credentials ~/client_secret.json
gog auth add <YOUR_EMAIL> --services gmail,calendar,drive,contacts,sheets,docs

執行後將獲得一串授權網址，請複製貼到本機瀏覽器中，完成 Google 登入與授權流程。

# 第三步：驗證與使用

回到 Control UI 聊天室，你可以直接用自然語言進行測試，例如：

> "Check my latest emails"
> "Show me my calendar for today"

如果你想在 VM 終端機直接下指令，也可以這樣用：

# 搜尋最新 5 封信
gog gmail search 'newer_than:1d' --max 5

# 查看今日日曆
gog calendar events primary --from $(date -I) --to $(date -I)

# 第四步：日常維護與注意事項

維持系統穩定運作，請留意以下幾點原則：

  • 保持 SSH 隧道開啟：只要需要訪問 Control UI，就必須保持本機的 SSH 隧道視窗開啟，關閉隧道等於斷開連線。
  • 安全性提醒：絕對不要在任何地方輸入您的 Google 密碼（僅限官方登入頁面）。Token 相當於您的服務鑰匙，請妥善保存。
  • 除錯起手式：如果遇到機器人不理人，請先查看服務狀態與日誌：
# 查看服務狀態
systemctl --user status clawdbot-gateway.service

# 查看即時日誌抓蟲
journalctl --user -u clawdbot-gateway.service -n 50

部署完成！現在你的雲端伺服器上已經有了一個完全整合 Google 生態的 AI 助理。🤖
